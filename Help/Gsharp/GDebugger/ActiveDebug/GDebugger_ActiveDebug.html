G#Debugger GUI - Active Debug tab

This tab will enable the G# Active Debug. It is a powerful way of debugging and testing methods without actual need to create and application. When enabling the active debug, the current active project (from which the debugger is lauched) will be scanned for classes and placed in the tree view control. Select a method in the tree view and the panel of the method will show up in the subpanel to the right. If a constructor method (the Create.vi) is selected and executed by the Run VI button, an object is created and the reference cached in the debugger. Then, another method of the class (or a parent class) could be selected to be executed. Control values could be edited in the subpanel before running it. It is also possible to open the front panel in the by either double clicking in the tree view or press the Open VI Panel button. The method VI could then be executed directly by using the standard Run button. Notice that you have to select the VI in the tree view before executing it, because the cached reference needs to be copied to the input reference of the method.

All created objects will of course show up in the objects tab and trace tab. To destroy the object, run the Destroy method or use the Garbage Collector button on the Objects tab. It is also possible to select any object in the object tree view and use this as the cached object.

Tab Control

Selects between Objects, Trace or Active Debug mode.

 

Enable Active Debug

Enables active debug mode. External create process will own the created objects.

 

DebugTree

All classes and public methods in the current active project.

 

Wait Until Done

Waits until the VI is finished until releasing control back to debugger.

 

Run VI

Runs the selected VI. Object reference is cached.

 

Abort VI

Aborts the selected VI (if running).

 

Open Panel

Opens the front panel of the selected VI.

 

External Create Process Running

Indicates if the background external create process is running.

 

Window Transparency

Transparency of the front panel window.

 

Window Behaviour

Selects if the debugger is on top of all VIs or not.

 

Cached Ref Valid

Each time a constructor is executed, the last reference is cached and will be copied to the new selected, opened or executed methods 'reference in'.

 

Object ID

Unique object id of the cached reference.

 

Cached Reference Class Name

Class of the cashed reference.

 

G# Version

Version of G#.

 

Object ID

Unique object identity.

 

Application Instance

Application instance of the object scope.

 

Owner

Owning top-level VI of the selected object. When this VI stops executing, LabVIEW will automatically garbage collect the object.
