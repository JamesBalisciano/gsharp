G#Object.lvclass:G#Object_GenericCreate.vi

Creates an instance of the class specified by the path. The path could either be directly to the constructor method, the .lvclass file or to the class folder. It is also possible to set the constructor input parameters by the "Control Names" and "Control Values" inputs.

Since this function runs the constructor with VI server, making the constructor "running top-level", it is necessary to use the external create event in order to let this VI create the needed object and become the owner of the queues and data value reference. However, if the constructor also contains user defined queues, notifiers etc., these will be garbage collected by LabVIEW when the constructor is finished executing. To avoid this, use the Default Constructor instead.

Be careful when using this VI when building executables, since paths must be correct and make sure that all needed VIs are included in the build.

Note: Will not work on RT systems.

class path

Specifies the path to the LabVIEW class.

respond to create event? (T)

If TRUE, this method will also respond to the create event. This will make the top-level calling VI the owner of the object and LabVIEW will not garbage collect the object until the top-level VI stops executing.

If FALSE, an other external process must respond to the G# create event.

keep constructor VI ref open? (T)

If true, the VI reference to the constructor VI will not be closed.
