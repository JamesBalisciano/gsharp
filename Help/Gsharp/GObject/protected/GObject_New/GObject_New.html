reference in

Object reference in.

user counter (T)?

If enabled and object is named, a counter of users occur. When the object is deleted, users will be decreased and object will not be killed until last user is zero.

object name

Name of the object. Not applicable if create method is 'create'.

create method

Selects how to create the object.

Create - Object name is ignored and a new object is always created.

Create or lookup existing - if object name is an empty string, a new object is always created. If object name is specified, a lookup of that object is performed and if existing this object will be returned, else a new named object will be created.

Lookup existing - tries to look up an existing object specified by the object name. If object name is not found or is an empty string, an error will occur.

no registration? (F)

If enabled, no registration of the object will occur. It will not show up in the debugger, all objects arrays etc. Garbage collector will not clean this object. It will not be affected by "External Create Process" either.
