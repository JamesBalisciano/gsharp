G#Object.lvclass:G#Object_EnableExternalCreate.vi

Sets if external create is enabled. This means that debugger or another process will respond to the G# event and create (and own) the objects (queues and dvrs) instead of the caller processes. It will also own whatever might be created in the overridden NewCallback method.

 external create enable (F)

Enables the external create (if 'set' is true).

set (F)

Selects between set or get the flag.

error in (no error)

error in can accept error information wired from VIs previously called. Use this information to decide if any functionality should be bypassed in the event of errors from other VIs.

Right-click the error in control on the front panel and select Explain Error or Explain Warning from the shortcut menu for more information about the error.

    		status

    status is TRUE (X) if an error occurred or FALSE (checkmark) to indicate a warning or that no error occurred.

    Right-click the error in control on the front panel and select Explain Error or Explain Warning from the shortcut menu for more information about the error.

    		code

    code is the error or warning code.

    Right-click the error in control on the front panel and select Explain Error or Explain Warning from the shortcut menu for more information about the error.

    		source

    source describes the origin of the error or warning.

    Right-click the error in control on the front panel and select Explain Error or Explain Warning from the shortcut menu for more information about the error.

enabled

Returns the state of the external create flag.

already enabled

Indicates if external create already was enabled.
